# Yukinu – Roadmap

This document outlines the current and future development plans for Yukinu.  
Yukinu is a learning project — priorities may change based on exploration needs.

## Completed

### Authentication & User Accounts

- User registration and login
- OAuth login (Google, Facebook)
- Update personal profile
- Secure session management

### Admin Capabilities

- Admin dashboard for user management
- View list of users
- Update user information and status

### Core Infrastructure

- PostgreSQL + Drizzle ORM setup
- tRPC API with Zod validation
- Basic monorepo structure with Turborepo
- Shared UI library using Tailwind + shadcn

## In Progress

- Vendor role management + onboarding
- Admin improved permissions and controls
- Basic product browsing UI
- Initial email notification flow (Resend)

## Planned

### Marketplace Features

- Vendors can add/store/manage products
- Shopping cart & checkout improvements
- Order lifecycle (processing → completed)
- Wishlist enhancement

### User Experience

- Product search and category filters
- Ratings & reviews system
- Recommended products / trending section

### Vendor Tools

- Sales and product insights dashboard
- Inventory management
- Messaging between buyer and vendor

### Payments & Transaction Flow

- Integrate external payment provider (e.g., Stripe)
- Order cancellation & refund handling

### 5️⃣ Security & Access Control

- Full role-based access control (RBAC)
- Admin tools for vendor approvals

---

## Experimental / Future Ideas

- CDN or edge delivery for product images
- SEO optimization for product pages
- Mobile app companion (React Native)
- AI product recommendations
- Event streaming architecture (e.g., Kafka)

---

## Not in Scope (For Now)

- Handling real financial/payment data directly
- Legal/regulated business features
- Real-world vendor verification or logistics

> Yukinu will remain a **non-commercial educational project**.
